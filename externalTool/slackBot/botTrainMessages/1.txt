### 1. User 서비스 실행하기  
---  
테스트를 위해서 User 서비스를 실행시켜야 합니다.  
자세한 가이드는 '프로젝트 입문 가이드 1 > 3. 서비스 실행시켜 보기'를 참조해 주시길 바립니다.  

1. .\compose 폴더에서 kafka를 동작시킵니다.  
```bash
docker-compose -f .\kafka-host.yml up
```  

2. .\source\user 폴더에서 다음 커맨드를 실행시켜서 User 서비스를 동작시키세요. 
```bash
.\command\hostRun.ps1
```

3. 정상적인 접속을 확인하세요.
```bash
http -v GET http://localhost:8082/sanityCheck
```

### 2. serverTester 프로그램 활용하기  
---  
1. ./externalTool/serverTester/ServerTesterRelease 경로로 이동합니다.  
2. ServerTester.exe 실행파일을 엽니다.

> * 선택 목록들에 대한 설명  
테스트 그룹: 의미 있는 단위로 각 테스트 그룹으로 묶여 있습니다. 특정 테스트 그룹을 선택해서 해당하 테스트 항목들을 볼 수 있습니다.  
테스트 항목: 각각의 HTTP 요청들이 묶여 테스트 항목을 형성하게 됩니다. 테스트 항목을 선택해서 해당는 HTTP 요청 항목들을 볼 수 있습니다.  
요청 내역: 테스트 항목에 포함된 각각의 HTTP 요청 스펙입니다. 각각을 클릭해서 '요청 로그'를 통해서 어떠한 요청을 하는지 확인할 수 있습니다.  

> * 출력 로그에 대한 설명  
검토 결과: 체크 항목들을 통과했는지의 여부를 출력합니다.  
요청 로그: 요청한 메소드, URL, HTTP 헤더, Body 내용을 확인합니다.    
응답 로그: 요청 결과로 나오는 응답 코드, Body 내용을 확인합니다.  

> * 버튼에 대한 설명  
테스트 항목 실행: 선택된 테스트 항목에 포함된 모든 요청 내역 위에서부터 순서대로 전부 실행시킵니다. 중간에 원하는 결과가 나오지 않을 경우 중지됩니다.  
단일 요청 실행: 특정 요청 내역 하나만은 실행시킵니다.  
초기화: 설정을 다시 불러오며, 모든 UI를 초기화시킵니다.  

3. 테스트할 항목이 있는 테스트 그룹을 선택합니다. User 테스트 그룹을 선택해 주시면 됩니다.  
4. '정상 작동 여부 체크용 엔드포인트들' 테스트 항목을 선택합니다.  
5. '통신 확인(SanityCheck)' 요청 항목을 선택합니다.  
6. '단일 요청 실행' 버튼을 누릅니다.  
7. 위의 사진과 같이 성공적으로 요청이 수행되었음을 확인합니다.  
8. '로그인 및 이름 변경 작동 여부 확인' 테스트 항목을 선택합니다.  
9. '테스트 항목 실행' 버튼을 누릅니다.
10. 위의 사진과 같이 모든 테스트 항목이 순차적으로 전부 통과되었음을 확인합니다.  
11. '초기화' 버튼을 눌러서 모든 UI 및 설정을 초기화시켜 봅니다.   

### 3. serverTester 프로그램에 테스트 항목 추가하기  
---  
serverTester에 테스트 항목을 추가하기 위한 방법을 알아봅니다.  
serverTester은 testItem 디렉토리에서 .xml 파일을 불러와서 테스트 항목을 구성시키기 때문에 이 부분을 수정하면 새로운 테스트 항목을 추가하거나 수정시킬 수 있습니다.  

1. .\testItem\user\BasicCheck.xml 파일을 엽니다.  

2. 다음과 같은 설정 내역들을 확인할 수 있습니다. 기본적인 문법들은 별로 어렵지 않으니, 실제 예제들을 보면서 학습해 보시길 바랍니다.  
> type="status", type="data" 속성을 가진 check 태그에 들어간 값은 정규식을 사용해서 포함 여부를 확인할 수 있습니다.
